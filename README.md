🏆 Multi-Agent-Restaurant-League

LLM 멀티 에이전트와 RAG를 활용한 리그전 방식의 맛집 추천 시스템

📖 프로젝트 소개

다양한 플랫폼에 흩어진 방대한 리뷰와 파편화된 별점 속에서 사용자가 겪는 검색 피로도를 줄이기 위해 기획된 프로젝트입니다. 사용자가 직접 식당들을 비교 분석할 필요 없이, AI가 객관적 지표와 리뷰 텍스트를 종합하여 검증된 최종 순위를 도출합니다.

특히 단순 평점 나열이나 확률 기반의 단일 추천이 아닌, '에이전트 간의 1:1 논쟁'이라는 게임화(Gamification) 요소를 도입하여 추천의 투명성과 신뢰성을 확보합니다.

✨ 핵심 기능 (Key Features)

맞춤형 데이터 수집: 세부 카테고리(예: 중식-자장면) 기반 식당 및 리뷰 데이터 수집.
AI 기반 1차 컷오프: 텍스트 분류 모델을 통한 정성/정량 데이터 분석으로 Top 10 식당 선별.

멀티 에이전트 논쟁 시뮬레이션: Top 10 식당 각각에 대변인 에이전트를 할당하여 LangGraph 기반 1:1 풀리그(Round-robin) 매칭 및 논쟁 수행.

심판(Judge) 판정 및 랭킹 보드: 심판 에이전트가 논쟁 스크립트를 분석해 승패를 판정하고 최종 승률(순위표)을 제공.

💡 기존 서비스와의 차별성

기존 네이버, 카카오 등 플랫폼의 AI는 자사 리뷰에 국한된 단일 식당 요약 브리핑만 제공하며, 다수의 식당을 교차 비교하는 상용 서비스는 부재합니다. 본 시스템은 식당별로 전담 에이전트를 부여해 서로의 장단점을 공격하고 방어하는 구조를 취하며, 실제 리뷰 데이터를 RAG로 연동하여 논리적 근거가 명확합니다. LangGraph를 이용해 전체 매치업과 리그 상태(State)를 중앙 관리함으로써, 단순 대화형 챗봇을 넘어선 고도화된 평가 시스템을 구현했습니다.

⚙️ 시스템 아키텍처 및 파이프라인 (Pipeline)

데이터 수집: 특정 메뉴에 해당하는 지역 식당의 리뷰 텍스트와 메타데이터(영업시간, 주차 등) 크롤링.

특징 분석 및 컷오프: 감성 분석 모델을 파이프라인에 적용해 리뷰 내 정성적 요소를 수치화하고 정량 데이터와 결합하여 Top 10 선정.

RAG 및 에이전트 세팅: 각 식당의 강점 데이터를 Vector DB에 주입하고, LangGraph 기반의 대변인 에이전트 할당.

리그전 논쟁 및 판정: 프롬프트 릴레이를 통해 1:1 논쟁을 진행하고, 심판(Judge) 에이전트가 승패를 판정하여 상태(State) 관리.

결과 제공: 크롤링 데이터 전처리 및 에이전트 논쟁에 소요되는 시간을 최소화하기 위해, 백그라운드 사전 연산을 거쳐 사용자에게 즉각적인 랭킹 보드 제공.
